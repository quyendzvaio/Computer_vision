Chế độ vẽ ROI (ROI Draw Mode)

Kích hoạt: Click nút "✎ ROI draw" trên toolbar → chuột chuyển sang chế độ vẽ
Thoát: Click lại nút hoặc nhấn ESC
Vẽ polygon mới
  • Click → thêm điểm
  • Double-click → đóng polygon
  • Mỗi ROI có màu riêng (A: đỏ, B: xanh, C: vàng)
  • Hiện label "Zone A", "Zone B" trên polygon
Chỉnh sửa ROI có sẵn
  • Kéo điểm → di chuyển vertex
  • Kéo cạnh → di chuyển cả polygon
  • Click phải → delete polygon
  • Luôn snap vào video frame

ROI Config — Lưu trữ

ROI polygon được lưu vào SQLite, load khi start:
Table: roi_config
  id         INTEGER PRIMARY KEY
  camera_id  TEXT NOT NULL        -- 'cam1'
  zone_name  TEXT NOT NULL        -- 'Zone A', 'Zone B'
  points     TEXT NOT NULL        -- JSON: [[x1,y1],[x2,y2],...]
  color      TEXT NOT NULL        -- '#ff0000'
  enabled    BOOLEAN DEFAULT 1

Table: cameras
  id         TEXT PRIMARY KEY     -- 'cam1', 'cam2'
  zmq_port   INTEGER             -- 5555, 5556
  source     TEXT                -- 'usb0', 'usb2'
  enabled    BOOLEAN DEFAULT 1

Alert Flow

QThread phát hiện violation: → emit alert_signal(violation_dict) → Main thread slot Main thread: 1. Ghi violation vào SQLite (async via QTimer) 2. Update status bar + alert counter 3. Flash đỏ border trên camera widget 4. Capture snapshot frame → lưu thumbnail 5. Broadcast alert qua WebSocket (FastAPI) 6. Nếu violation nghiêm trọng → system tray notification
Alert dedup (Cooldown):
Mỗi loại violation cho 1 person có cooldown riêng. Ví dụ: Nếu person A đã alert NO_VEST, không alert lại trong 30s cho person A. Tránh spam.
SQLite Schema — violations:
Table: violations
  id           TEXT PRIMARY KEY
  camera_id    TEXT
  type         TEXT           -- 'PERSON_IN_ZONE', 'NO_HELMET', 'NO_VEST', 'NO_BOOT'
  severity     TEXT           -- 'HIGH', 'MEDIUM', 'LOW'
  zone_name    TEXT           -- NULL nếu là PPE
  bbox_json    TEXT
  thumbnail    TEXT           -- path to saved image
  timestamp    DATETIME DEFAULT CURRENT_TIMESTAMP
  acknowledged BOOLEAN DEFAULT 0

FastAPI Web — Secondary View

  • Không broadcast frame real-time qua WS — tránh tốn CPU encode
  • REST API: GET /violations, GET /cameras, GET /roi/{camera_id}
  • WebSocket: Chỉ push alert text (type, camera_id, timestamp)
  • History page: Danh sách violation có phân trang + filter
  • Admin page: Xem/sửa ROI config, camera config
FastAPI chạy trong thread riêng:
Ủy quyền asyncio event loop vào thread riêng, không block main PyQt loop. Dùng QThread mới chạy uvicorn.
