Design — Data Flow & Frame Budget

Chiến lược frame pacing giữa các thread để tối ưu latency

Edge → GPU: ZeroMQ PUB/SUB

Edge device gửi JPEG encoded frame (không phải raw BGR) qua ZMQ:
USB Cam1 @ 30fps → JPEG encode (~2ms) → ZMQ PUB topic "cam1" :5555
USB Cam2 @ 30fps → JPEG encode (~2ms) → ZMQ PUB topic "cam2" :5556
Tại sao JPEG?
Raw BGR 640×480 = 921KB/frame. JPEG 640×480 ~50KB/frame → 18x nhỏ hơn. Latency network giảm tương ứng. GPU machine decode JPEG (~1ms) vẫn nhanh hơn truyền raw frame.

Frame Budget — Mỗi QThread tự điều tiết

Mỗi QThread có vòng lặp chính với deadline cố định:
FRAME_BUDGET = 33ms  # ~30fps target

Mỗi iteration:
  t0 = time()
  frame = recv() or skip_queue
  objects = infer(frame)     # YOLOv8n GPU: ~10-15ms
  check(objects)             # ROI or PPE: ~1-2ms
  overlay = draw(objects)    # ~2ms
  emit_signal(overlay)       # signal/slot: ~0.1ms

  elapsed = time() - t0
  if elapsed > FRAME_BUDGET:
    skip = next_frame        # skip queue để bắt kịp
    print(f"[Cam1] Slow frame: {elapsed:.0f}ms, skip next")
Chiến lược skip:
Nếu 1 frame chậm (vd: nhiều object → detect lâu hơn) → skip frame kế tiếp. Không dùng queue dồn frame — queue dài gây tăng latency tích lũy. Chỉ giữ latest frame (queued) để luôn xử lý frame mới nhất.

CAM1 Pipeline — ROI Inside

Frame → YOLOv8n detect person → foot point (x_center, y2)
  → Với mỗi person:
    → Với mỗi ROI polygon:
      → point_in_poly(foot_point, roi)?
    → Nếu inside bất kỳ ROI nào:
      → Nếu chưa trigger timeout cho person này → ALERT "person_in_zone"
      → Set cooldown (tránh alert trùng trong N giây)
Chi tiết: "person trong vùng cấm" là violation. Dùng foot point (đáy bbox) thay vì center để tránh trường hợp phần thân trên ở trong vùng cấm nhưng chân ở ngoài.

CAM2 Pipeline — PPE Detection

Frame → YOLOv8n detect person + helmet + vest + boot
  → Với mỗi person (bbox P):
    → helmet_found = any(helmet_bbox & P có IoU > 0.3)
    → vest_found = any(vest_bbox & P có IoU > 0.2)
    → boot_found = any(boot_bbox & P có IoU > 0.1)
    → Nếu thiếu cái nào → ALERT "NO_HELMET" / "NO_VEST" / "NO_BOOT"
Lưu ý IoU threshold khác nhau:
Helmet (mũ) = overlap nhiều với phần đầu → 0.3. Vest (áo) = phủ thân → 0.2. Boot (giày) = nhỏ, ở dưới cùng → 0.1. Có thể cần tuning.

Main Thread — Render & Alert

QThread emit signal → Main thread slot nhận frame + objects:
  • QLabel/QWidget: Hiển thị 2 camera side-by-side hoặc tab
  • QPainter overlay: Vẽ bbox + label + ROI polygon
  • ROI drawing mode: Mouse click → add point, double-click → close polygon, drag → move point
  • Alert popup: QSystemTrayIcon + notification + splash frame với ảnh violation
  • FastAPI async: Thread chạy song song với main loop, serve web secondary

Latency Budget Dự Kiến

Giai đoạn Latency
Edge: capture + JPEG encode ~5ms
ZMQ network (LAN) ~1ms
GPU: receive + JPEG decode ~2ms
YOLOv8n inference (GPU) ~12ms
ROI/PPE check ~2ms
Overlay draw ~2ms
Tổng (end-to-end) ~24ms
≈ 40fps throughput. Với skip frame khi quá tải, đảm bảo latency ổn định dưới 33ms (30fps) cho cả 2 camera.
