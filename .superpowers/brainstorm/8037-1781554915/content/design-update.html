Design Update — Sửa theo Senior Review

4 thay đổi chính để tối ưu latency + boots ~60%

🔧 Thay đổi 1: PPE Pipeline — Crop + Classifier thay vì YOLOv8 multi-class

Mới: YOLOv8n detect person → crop theo keypoint estimation từ bbox → classifier nhỏ
YOLOv8n detect person (640×480) → bbox (x1,y1,x2,y2)

Với mỗi person bbox:
  ├─ Head crop:   [x1:x2, y1:y1+0.2*h]   → upscale 224×224 → MobileNetV3 → helmet? (60-70%)
  ├─ Torso crop:  [x1:x2, y1+0.2*h:y2-0.3*h] → upscale 224×224 → MobileNetV3 → vest? (65-75%)
  └─ Feet crop:   [x1:x2, y2-0.15*h:y2]  → upscale 224×224 → MobileNetV3 → boots? (55-65%)

Mỗi classifier: MobileNetV3-small (ONNX) ~2-5ms trên GPU
Tổng: 15ms detect + 3×4ms crop + 3×5ms classify ≈ ~30ms/frame
Boots ~60%: Crop vùng chân từ bbox person (15% chiều cao dưới cùng), upscale lên 224×224. Boots chiếm ~80×80px trong crop → classifier thấy rõ. Accuracy 55-65%.
So với multi-class YOLOv8 (15-25%): Cải thiện ~3x nhờ upscale.
🔴 Đánh đổi: +10-15ms/frame so với YOLOv8 multi-class. Bù lại bằng skip frame strategy: chỉ chạy classifier mỗi 2-3 frame (~10fps cho PPE). Frame không có motion → skip hoàn toàn.

🔧 Thay đổi 2: Raw BGR thay vì JPEG

Cũ: JPEG encode (edge) → ZMQ → decode (GPU) — ~3ms/frame
Mới: Raw BGR → ZMQ → decode 0ms
LAN 1Gbps: Raw 640×480×3×30fps = 27MB/s — chỉ 2.7% bandwidth. Hoàn toàn chấp nhận được.

🔧 Thay đổi 3: Web preview 1fps + ZMQ HWM=2

Web preview 1fps
Main thread push frame có overlay vào queue → FastAPI thread lấy mỗi 1s → WS broadcast.
Tốn ~2ms CPU mỗi lần encode, không đáng kể.
ZMQ_SNDHWM = 2
Edge device chỉ queue tối đa 2 frame trong kernel buffer. Frame cũ bị drop tự động. Tránh OOM khi GPU xử lý chậm.

🔧 Thay đổi nhỏ khác

  • USB device path — Dùng /dev/v4l/by-id/ thay vì index cứng
  • QThread crash recovery — try/except run() → restart ≤5 lần/phút
  • FastAPI lifecycle — uvicorn.Server với shutdown event
  • Auto-save ROI — Mỗi lần vẽ xong tự động lưu SQLite
