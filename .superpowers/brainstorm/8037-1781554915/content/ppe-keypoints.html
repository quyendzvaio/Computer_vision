CAM2 — PPE Detection via YOLO-Pose Keypoints

Dùng keypoints skeleton thay vì bounding box overlap để detect thiếu bảo hộ

Ý tưởng: Keypoint-based PPE detection

Thay vì detect "người" + "mũ/áo/giày" riêng rẽ rồi kiểm tra overlap (dễ sai khi object nhỏ), dùng YOLO-Pose để lấy khung xương người, sau đó phân tích từng vùng trên cơ thể:

YOLO-Pose output: 17 keypoints (COCO format)
├─ 0: nose ├─ 5: left_shoulder ├─ 10: right_wrist
├─ 1: left_eye ├─ 6: right_shoulder ├─ 11: left_hip
├─ 2: right_eye ├─ 7: left_elbow ├─ 12: right_hip
├─ 3: left_ear ├─ 8: right_elbow ├─ 13: left_knee
└─ 4: right_ear ├─ 9: left_wrist ├─ 14: right_knee
├─ 15: left_ankle
└─ 16: right_ankle
Phân tích:
• Vùng đầu (nose + ears + eyes) → crop → classifier nhỏ: có mũ bảo hộ không?
• Vùng thân trên (shoulders → hips) → crop → classifier: có áo bảo hộ không?
• Vùng chân (ankles + knees) → crop → classifier: có giày bảo hộ không?
Ưu điểm so với bbox overlap:
• Keypoints cho biết CHÍNH XÁC vị trí từng bộ phận
• Crop đúng vùng cần kiểm tra → classifier nhẹ hơn
• Không bị nhiễu khi mũ/áo/giày nhỏ trong bbox tổng
• Có thể threshold theo confidence từng keypoint

Phân luồng xử lý CAM2

1. QThread đọc frame từ USB Camera2
2. YOLO-Pose inference → 17 keypoints per person
3. Với mỗi person có keypoints:
├─ Head crop (nose ± shoulder_width) → Classifier: helmet?
├─ Torso crop (shoulders → hips) → Classifier: vest?
└─ Feet crop (ankles ± offset) → Classifier: boots?
4. Nếu bất kỳ classifier nào → "NO" → trigger violation
5. Overlay kết quả lên frame → MediaMTX → Dashboard
6. Ghi violation vào SQLite

Latency Optimization Strategy

Frame pipeline
Frame N → YOLO-Pose → crop → classify(small CNN)
Frame N+1 → (skip classify nếu pose conf thấp)
Frame N+2 → process đầy đủ
👉 Motion skip: Bỏ qua frame nếu không có chuyển động
QThread model
QThread1: capture + inference + draw
(tất cả trong 1 thread, queue-based)
👉 Zero-copy: Frame buffer reuse
👉 No IPC: shared memory loại bỏ
👉 Direct-to-GPU: OpenGL widget
