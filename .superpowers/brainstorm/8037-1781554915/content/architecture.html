Kiến trúc đề xuất — CV Safety Monitor v2

Chia làm 2 pipeline độc lập, mỗi camera một bài toán riêng

Tổng quan luồng dữ liệu

📷 CAM1 — Phát hiện ra khỏi vùng ROI
USB Cam1 → PyQt Capture (QThread) → YOLOv8 Detector → ROI Exit Check → MediaMTX RTSP → Dashboard/App
↕ SharedMemory
Alert Pipeline → SQLite
📷 CAM2 — Phát hiện trang bị bảo hộ (PPE)
USB Cam2 → PyQt Capture (QThread) → YOLO-Pose → Keypoint Analyzer → MediaMTX RTSP → Dashboard/App
│ ↓
│ PPE Classifier (head/hands/feet)
│ → Alert Pipeline → SQLite
🖥️ PyQt5 Desktop App (Main Process)
├─ QThread: Capture + Inference (CAM1)
├─ QThread: Capture + Inference (CAM2)
├─ MediaMTX Embedded: RTSP relay cho cả 2 cam
├─ Real-time overlay + alert popup
└─ WebSocket bridge → Dashboard Web (optional)

So sánh kiến trúc cũ vs mới

Hiện tại (Multi-process)
P1 (thread): capture_frame → overlay → WS
P2 (process): inference → SharedMemory
Async: dashboard server
❌ Shared memory IPC overhead
❌ Process sync complexity
❌ 1 pipeline cho mọi camera
Mới (PyQt5 + QThread)
QThread 1: CAM1 capture + inference + ROI
QThread 2: CAM2 capture + inference + PPE
Main thread: PyQt UI + MediaMTX relay
✅ No IPC — shared memory in-process
✅ Pipeline độc lập, không blocking
✅ Real-time UI overlay native
