Person INSIDE ROI vs Person LEAVE ROI — Cái nào dễ hơn?

Phân tích độ phức tạp, latency và độ tin cậy

So sánh trực tiếp

✅ INSIDE ROI — Dễ
Mỗi frame:
detect → foot point → point_in_poly?
  • 1 frame là đủ — không cần tracking
  • Không có state — stateless, dễ debug
  • Không có ID swap — không tracker, không false positive từ ID
  • Latency thấp nhất — chỉ check geometry
  • Mất mát: skip frame không ảnh hưởng
Phù hợp: ROI = vùng cấm → xuất hiện trong vùng cấm là vi phạm
❌ LEAVE ROI — Khó
Mỗi frame:
detect → track → check transition inside→outside
  • Cần 2+ frame — phải so sánh frame trước-sau
  • Có state tracking — ID, vị trí, trạng thái
  • ID swap = false alarm — 2 người đi qua nhau tưởng "rời zone"
  • Skip frame = mất transition — nếu skip đúng lúc người rời zone
  • Occlusion = mất track → treo state vĩnh viễn
Phù hợp: ROI = vùng an toàn → rời khỏi vùng an toàn là vi phạm

Tại sao INSIDE ROI dễ hơn nhiều?

Với accuracy target 65% và latency là ưu tiên:

  • INSIDE ROI: Chỉ cần detect person đúng ~65% frame. Skip frame 50% vẫn OK. Nếu frame này detect được person trong ROI → alert. Nếu frame sau skip → không sao, alert đã được gửi từ frame trước.
  • LEAVE ROI: Cần tracking ổn định qua nhiều frame. Skip frame có thể mất transition. Nếu accuracy 65% → tracking càng tệ hơn (lỗi chồng lỗi). False positive rate sẽ cao hơn nhiều.

Kết luận: INSIDE ROI đơn giản hơn, latency thấp hơn, ít false positive hơn. LEAVE ROI chỉ nên chọn nếu use case bắt buộc phải detect "rời khỏi vùng an toàn".

🚩 Vấn đề kiến trúc: Edge device + Máy GPU

Đây là điểm chưa hợp lý nhất trong thiết kế hiện tại

Bạn nói có 1 edge device (gắn camera) và 1 máy GPU (chạy inference). Kiến trúc PyQt + QThread đề xuất ở trên chỉ đúng khi mọi thứ chạy trên cùng 1 máy.

Với 2 máy riêng, có 2 hướng:

Hướng A: Edge capture, GPU inference
Edge: PyQt capture → gửi frame qua mạng
GPU: nhận → inference → ROI/PPE → kết quả
Network latency: 1-5ms (LAN)
✅ GPU tận dụng tối đa
❌ Phải có network protocol
Hướng B: GPU machine chạy tất cả
Edge: chỉ relay USB camera → GPU machine qua USB-over-IP hoặc RTSP
GPU: PyQt + QThread + inference + UI
✅ Kiến trúc đơn giản nhất
❌ USB-over-IP có thể không ổn định
