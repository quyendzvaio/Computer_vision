Design — Error Handling & Testing Strategy

Xử lý lỗi edge device, crash, network mất kết nối

Error Handling — Edge Device

Camera disconnect
USB camera rút ra → cv2.VideoCapture trả False → retry 5 lần (1s interval) → log lỗi → chờ reconnect
ZMQ send failure
GPU machine restart → ZMQ PUB vẫn gửi (ZMQ queue ở edge) → khi GPU online lại, auto-receive. Không crash.
Frame encode lỗi
JPEG encode thất bại → skip frame, không retry. Vì frame kế tiếp sẽ đến ngay (30fps).
Edge auto-restart
systemd service: Restart=always. Script đơn giản, start lại dưới 1s.

Error Handling — GPU Machine

ZMQ SUB timeout
Không nhận được frame từ edge trong N giây → hiển thị "Disconnected" overlay trên camera widget → retry.
Inference crash
ONNX session lỗi → log + fallback: vẫn hiển thị frame raw (không detect). Không crash app. User thấy "Detection offline" badge.
GPU OOM
Nếu 2 model instance cùng OOM → catch exception → giảm batch size → restart model session. Nếu vẫn lỗi → fallback CPU mode.
PyQt main loop không block
Mọi xử lý nặng đều trong QThread. Main thread chỉ render + events. Nếu QThread crash → signal error → main thread restart thread.

Testing Strategy

1. Unit tests (pytest, không cần camera)
  • ROIChecker: point_in_poly với polygon chuẩn, cạnh, ngoài
  • PPEChecker: IoU calculation với các bbox overlap khác nhau
  • FrameBudget: Mô phỏng frame chậm, kiểm tra skip logic
  • AlertManager: Cooldown dedup, duplicate suppression
2. Integration tests (cần GPU)
  • Inference pipeline: Frame → YOLOv8 → objects → output format
  • ZMQ round-trip: PUB frame → SUB receive → decode
  • Full pipeline CAM1: Frame → detect → ROI → alert
  • Full pipeline CAM2: Frame → detect → PPE → alert
3. Edge device simulation (test không cần edge HW)
  • Script phát lại video file (.mp4) qua ZMQ PUB giả lập edge
  • Test GPU machine với "edge simulator" trên cùng máy
  • Skip frame, disconnect, latency spike đều có thể mô phỏng

Edge Device Failure Scenarios — Flowchart

Edge mất kết nối LAN: ├─ Edge side: ZMQ PUB vẫn chạy, queue frame ở kernel buffer ├─ GPU side: ZMQ SUB timeout (ko nhận frame >3s) │ ├─ → Overlay "DISCONNECTED" trên widget │ ├─ → Log event │ └─ → Tiếp tục loop, chờ reconnect └─ Khi LAN back online: ├─ ZMQ auto-deliver frame mới nhất └─ GPU: render lại bình thường trong ~1 frame
