Vai trò từng công nghệ trong dự án

Tại sao cần PyQt5, QThread, MediaMTX — và tại sao từng cái phù hợp

1. PyQt5 — GUI Framework

Thay thế: FastAPI Web Dashboard (hiện tại)
Vai trò: Tạo cửa sổ desktop native để hiển thị video real-time + overlay detection + ROI drawing + alert popup
Tại sao chọn: Qt là thư viện GUI mạnh nhất Python. Có QPainter/QGraphicsView để vẽ overlay siêu nhanh trực tiếp lên frame, không cần encode/decode base64 như WebSocket. Vẽ ROI polygon kéo thả mượt.

2. QThread — Đa luồng native

Thay thế: multiprocessing.Process + SharedMemory IPC (hiện tại)
Vai trò: Mỗi camera chạy trong 1 QThread riêng: capture → inference → draw → relay, tất cả trong cùng tiến trình
Tại sao chọn: QThread giao tiếp với main thread qua signal/slot, không cần shared memory, không cần pickle, không cần multiprocessing sync. Dữ liệu frame là con trỏ trong cùng address space → zero-copy. Giảm latency đáng kể so với IPC.

3. MediaMTX — RTSP Relay Server

Thay thế: WebSocket broadcast preview (hiện tại)
Vai trò: Nhận frame từ PyQt → publish thành RTSP stream → các client khác (VLC, Dashboard Web, mobile app) có thể xem
Tại sao chọn: MediaMTX là RTSP server siêu nhẹ (Go, single binary). Cho phép nhiều client subscribe 1 stream mà không ảnh hưởng đến latency của pipeline chính. Tách biệt viewing khỏi processing — processing chạy real-time trong QThread, MediaMTX chỉ relay.

So sánh architecture cũ → mới

Hiện tại (multi-process + WS)
Luồng frame:
Capture (P1 thread)
→ write SharedMemory → P2 đọc
→ inference → write result SharedMemory
→ P1 đọc → overlay → base64 encode
→ WebSocket → browser render
⚠️ 3 lần copy frame
⚠️ base64 encode/decode
⚠️ Python GIL + multiprocessing overhead
Mới (PyQt + QThread + MediaMTX)
Luồng frame:
QThread: capture → inference → overlay
→ emit signal → PyQt widget render
(cùng process, cùng memory space)
├── Gửi copy frame → MediaMTX → RTSP
└── (nếu có alert) → SQLite + WebSocket
✅ 0 IPC, 0 base64
✅ Zero-copy trong pipeline
✅ Qt signal/slot thread-safe native
✅ Viewing tách biệt khỏi processing

Optional: Giữ Dashboard Web?

PyQt là primary UI. Dashboard Web có thể giữ lại như secondary view:

  • PyQt render native → MediaMTX RTSP → Web dashboard đọc RTSP và hiển thị
  • WebSocket vẫn dùng để push alert text cho web
  • Không cần base64 frame trên WebSocket nữa → giảm tải đáng kể
